## **What is the White-Label Desktop App?**

  


The White-Label Desktop App gives agencies a way to create and distribute a branded desktop application for their customers. Instead of relying only on the standard HighLevel or LeadConnector desktop app experience, agencies can customize the desktop app’s visual identity so it better reflects their own brand.

  


The Desktop App Customizer is available from the Desktop App tab in the Agency Dashboard. Agencies can apply branding updates, preview the app in real time, submit the app for build, and then download the app or copy generated access links once the build is complete.

  

    
    
    **Note:** The White-Label Desktop App is separate from the standard HighLevel and LeadConnector desktop apps. Use it when you want to customize and distribute a branded desktop app for your customers. For the standard desktop app, see [Download HighLevel & LeadConnector Desktop App for Windows, macOS & Linux](<https://help.gohighlevel.com/support/solutions/articles/155000006704-download-highlevel-leadconnector-desktop-app-for-windows-macos-linux?utm_source=chatgpt.com>).

## **Key Benefits of the White-Label Desktop App**

  


A branded desktop app helps agencies create a more consistent customer experience while giving users a focused way to access HighLevel from their desktop. These benefits are especially useful for agencies that want their software experience to feel cohesive across login pages, apps, and customer onboarding materials.

  


  * **Fully branded desktop experience:** Agencies can customize the app name, app icon, theme, login screen title, subtitle, full logo, background image, and placeholder images to align the desktop app with their brand.  
  

  * **Cross-platform availability:** Agencies can create branded desktop apps for macOS, Windows, and Linux.  
  

  * **Live preview before building:** The Desktop App Customizer shows changes in real time so agencies can review the app’s look and feel before submitting it for build.  
  

  * **Automated builds:** After the Build App action is triggered, HighLevel starts the build process and generates application links when the build is complete.  
  

  * **Streamlined distribution:** Agencies can download the completed app or copy the generated access links within the timeframe shown in HighLevel.  
  

  * **More focused desktop experience:** A desktop app gives users a dedicated workspace outside the browser, helping reduce distractions and improve day-to-day workflow efficiency.

## **How To Setup the White-Label Desktop App**

  


A clear setup process helps agencies review branding, submit the app correctly, and distribute the finished desktop app with confidence. Complete each step in order so the generated app reflects the intended branding before it is shared with customers.

  


### **Step 1: Open the Desktop App Customizer**

Log in to your Agency Dashboard and open the Desktop App tab. This is where you can configure the White-Label Desktop App, review branding changes in the live preview, and submit the app for building.

  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155073831464/original/7oGdtvHRe6PrzZUlXJLLM_VvjAhnnGG3LQ.png?1781677640)  
  


### **Step 2: Configure the Desktop App**

Add the desktop app branding details you want customers to see. These settings control the app’s visual identity, login screen content, and brand presentation inside the White-Label Desktop App.

  


  * **Theme:** Choose a preset theme or create a custom theme for the White-Label Desktop App.  
  

  * **App Icon:** Upload the app icon you want customers to see when using the desktop app.  
  

  * **App Name:** Enter the app name shown in the title bar and sign-in button. The app name cannot be changed while the app is being built.  
  

  * **Title:** Add the large heading that appears on the login screen.  
  

  * **Subtitle:** Add the supporting text that appears below the title on the login screen.  
  

  * **App Full Logo:** Upload the full logo for the app. The image is scaled to 512 x 128.  
  

  * **Background Image:** Upload the background image for the desktop app experience.  
  

  * **Placeholder Images:** Upload placeholder images to enhance the app’s appearance.


  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155073832061/original/2KReE4L9jRMhBvWLOgPBc7TF-LjIoZWSfg.gif?1781678376)  
  


### **Step 3: Review the Live Preview**

Review the live preview to confirm the desktop app reflects the selected branding settings. The preview updates as customizations are applied, allowing agencies to visualize the final app before submitting it for build.

  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155073832131/original/O3_lPbfpw81X6jTJNi8NW62jm8M1JiKHiw.png?1781678424)  
  


### **Step 4: Save the Configuration**

Save the configuration once the branding settings are complete. Saving prepares the selected theme, app name, icon, title, subtitle, full logo, background image, and placeholder images for the build process.

  


### **Step 5: Submit the App for Build**

After saving the configuration, submit the app for building. Once the Build App action is triggered, HighLevel automatically starts the build process.

  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155073832192/original/omtlz-eI0IEkD_tcqGYlD6-Cy8MsmqLnvw.png?1781678492)  


  


### **Step 6: Download, Copy, and Share Access Links**

Once the build is complete, HighLevel generates the corresponding application links. Agencies can download the app or copy the access links within the timeframe shown in HighLevel, then share the appropriate links with users.  
  


These links can be included in:

  * Onboarding materials
  * Client portals
  * Documentation
  * Customer support responses

## **Using Desktop App Branding With a Whitelabel Domain**

  


Desktop app branding controls how the installed White-Label Desktop App looks, including the app name, icon, theme, logo, and login screen visuals. A whitelabel domain controls the branded web login URL customers use when accessing HighLevel through a browser.

  


These settings are configured separately, but using both helps create a more consistent branded experience for customers. Use the Desktop App Customizer to brand the desktop app, and configure the whitelabel domain separately for the branded web login experience.

  


For setup instructions, see [How to Set Up a Whitelabel Domain for the Desktop Web App](<https://help.gohighlevel.com/support/solutions/articles/48000982207-how-to-set-up-a-whitelabel-domain-for-the-desktop-web-app?utm_source=chatgpt.com>).

## **Frequently Asked Questions**

  


**Q: Where do I find the Desktop App Customizer?**  
The Desktop App Customizer is available from the Desktop App tab in the Agency Dashboard.

  


**Q: Which platforms are supported?**  
The White-Label Desktop App is available for macOS, Windows, and Linux.

  


**Q: Can I preview the app before building it?**  
Yes. The Desktop App Customizer includes a live preview so you can review branding changes in real time before submitting the app for build.

  


**Q: What happens after I click Build App?**  
HighLevel automatically starts the build process. Once the build is complete, application links are generated and made available for access.

  


**Q: How do I share the completed desktop app with users?**  
After the build is complete, download the app or copy the generated access links from the Desktop App tab. These links can be shared with customers through onboarding emails, documentation, or a client portal.

  


**Q: Are Mac App Store and Microsoft Store listings available now?**  
The release note lists Mac App Store and Microsoft Store listing as a future enhancement. For now, use the generated download or access links provided after the build is complete.

  


**Q: Is the whitelabel domain configured inside the Desktop App Customizer?**  
No. The Desktop App Customizer controls the desktop app’s branding and build settings. A whitelabel domain is configured separately for the branded desktop web app login experience.
