## **What This Feature Does**

  


You can now:

  * **Duplicate a pipeline** within the same sub-account

  * **Copy a pipeline to multiple sub-accounts**

  * Preserve **pipeline stages and reporting visibility settings**


  


This is especially useful for agencies that want to standardize sales pipelines across multiple client accounts.

  


# **Permissions Required**

  


The ability to duplicate pipelines or copy them to other sub-accounts depends on your user permissions.

  


**Duplicate a Pipeline**

  * Any user with **Edit access to the pipeline** can duplicate it within the same sub-account.


  


**Copy Pipeline to Sub-Accounts**

  * Only **Agency Admins** can copy pipelines to other sub-accounts.


  


If you do not see the **Copy to sub-accounts** option in the pipeline menu, your user role likely does not have the required agency-level permissions.

* * *

# **How to Duplicate a Pipeline**

  


Duplicating a pipeline creates a copy within the **same sub-account**.

  


### **Step 1: Navigate to Pipelines**

  


Go to:

  


**Opportunities → Pipelines**

  


You will see a list of all pipelines available in the account.

* * *

### **Step 2: Open Pipeline Actions**

  


Find the pipeline you want to duplicate and click the **three-dot menu** on the right.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066782919/original/mVSFMcHCSQGnuHWxyC2fpG5gjt8P4SU8qA.png?1773316285)

  


Select **Duplicate**.

* * *

### **Step 3: Configure the Duplicate**

  


A **Duplicate Pipeline** window will appear.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066782966/original/YEfmEkPdAYLzgyuUH5fTncRW61a9v3asrQ.png?1773316300)

  


Update the following:

  


**Pipeline Name**

Enter a new name for the duplicated pipeline. A default name will appear with “(copy)” added.

  


The following elements will be copied automatically:

  * Pipeline stages

  * Chart visibility settings

  * Smart tag colors

  * Pipeline permissions


  


The following **will NOT be copied** :

  * Opportunity records

  * Automations

  * Reports

  * Custom fields


  


Click **Duplicate** to create the new pipeline.

* * *

### **Step 4: Confirm Creation**

  


The duplicated pipeline will appear in the pipeline list.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066782983/original/v-HHFDyNLCuW6jYyPzH8MwEixqaVow1KQA.png?1773316314)

  


You can now edit the pipeline or start using it immediately.

* * *

# **How to Copy a Pipeline to Sub-Accounts**

  


This feature allows **Agency Admins** to copy a pipeline to multiple sub-accounts at once.

  


### **Step 1: Open Pipeline Actions**

  


Go to:

  


**Opportunities → Pipelines**

  


Find the pipeline you want to copy and open the **three-dot menu**.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066783001/original/LWe5T1PIQD7l0-Zyemzm5cvQJjQjKMN8NQ.png?1773316329)

  


Select **Copy to sub-accounts**.

* * *

### **Step 2: Select Destination Sub-Accounts**

  


The **Copy to Sub-Accounts** window will appear.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066783007/original/M4PO_Sz-vc2hJH3_s7VrPTyu4aByfC-atA.png?1773316341)

  


Enter the following:

  


**Pipeline Name**

You can keep the same name or change it.

  


**Select Sub-Accounts**

Choose one or more sub-accounts where the pipeline should be created.

* * *

### **Step 3: Copy the Pipeline**

  


Click **Copy**.

  


The pipeline will be created in each selected sub-account.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066783024/original/csVsAzQOCZDwhaQSP7PZMkiyt9Bo6WY4Lw.png?1773316351)

* * *

# **What Gets Copied**

  


When duplicating or copying a pipeline, the following are included:

  * Pipeline stages

  * Stage order

  * Chart visibility settings


* * *

# **What Does NOT Get Copied**

  


The following items are **not transferred** when duplicating or copying pipelines:

  * Opportunity records

  * Custom fields

  * Pipeline permissions (when copying to other accounts)

  * Smart tags

  * Automations

  * Reports


  


These elements remain specific to each sub-account.

* * *

# **Best Practices**

  


**Use pipeline duplication for testing**

Duplicate a pipeline before making major changes to avoid affecting active sales processes.

  


**Standardize pipelines for agencies**

Create a master pipeline and distribute it to client accounts using the copy feature.

  


**Rename pipelines clearly**

If duplicating pipelines frequently, use clear naming conventions to avoid confusion.
