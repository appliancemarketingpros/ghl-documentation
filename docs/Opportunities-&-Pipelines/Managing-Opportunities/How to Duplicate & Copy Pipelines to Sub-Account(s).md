# **What is Duplicate & Copy Pipelines to Sub-Account(s)?**

  


Duplicate Pipelines and Copy Pipelines to Sub-Accounts help you reuse an existing opportunity pipeline instead of rebuilding it manually. This makes it easier to standardize sales processes, save setup time, and keep pipeline structures consistent across accounts. Teams can use duplication to create a version of a pipeline in the same sub-account, or use copy to send a pipeline to one or more sub-accounts.

  


Managing similar sales pipelines across multiple sub-accounts often requires repetitive manual setup. With this feature, agency admins can quickly copy pipelines to other sub-accounts, and users with the right access can duplicate pipelines within the same sub-account. This saves time and helps maintain consistency across accounts.

* * *

## **Key Benefits of Duplicate & Copy Pipelines to Sub-Account(s)**

  


  * **Faster setup:** Reuse an existing pipeline instead of creating stages one by one.  
  


  * **Improved consistency:** Keep sales processes aligned across sub-accounts.  
  


  * **Simpler onboarding:** Roll out proven pipeline structures to new sub-accounts more quickly.  
  


  * **Less manual work:** Eliminate repetitive configuration when managing multiple accounts.  
  


  * **Safer testing:** Duplicate a pipeline in the same sub-account to test changes before wider use.


* * *

## **Duplicate vs Copy to Sub-Accounts**

  


Understanding the difference between these two actions helps you choose the right workflow for your goal. One option is designed for reuse inside the same sub-account, while the other is designed for distribution across multiple sub-accounts.

  


  * **Duplicate:** Creates a copy of an existing pipeline inside the same sub-account.  
  


  * **Copy to sub-accounts:** Sends a copy of the pipeline to one or more different sub-accounts.  
  


  * **Use Duplicate when:** You want to test a variation, create a similar pipeline, or avoid rebuilding stages in the same account.  
  


  * **Use Copy to sub-accounts when:** You want to standardize pipeline structures across client or location accounts.

## **Permissions Required**

  


Access to duplicate or copy pipelines depends on the user's role and pipeline permissions. Reviewing these requirements first can help explain why a user may not see one or both actions in the pipeline menu.

  


  * **Duplicate pipeline:** Available to users with **Edit access** to that pipeline.  
  


  * **Copy to sub-accounts:** Available to **Agency Admins only**.


* * *

## **What Does and Doesn't Get Copied**

  


Knowing what transfers with the pipeline helps set the right expectations before you duplicate or copy it. This is especially important when working across sub-accounts, where some settings do not move with the pipeline.

  


When you **duplicate a pipeline within the same sub-account** , the following are copied:

  


  * Pipeline stages

  * Pipeline permissions

  * Colored smart tags

  * Chart visibility settings


  


When you **copy a pipeline to other sub-accounts** , the following are copied:

  


  * Pipeline stages

  * Chart visibility settings


  


The following items are **not copied** :

  


  * Records

  * Automations

  * Reports

  * Custom fields


  


When copying to other sub-accounts, the following are also **not copied** :

  


  * Pipeline permissions

  * Colored smart tags

## **How to Duplicate a Pipeline in the Same Sub-account**

  


Duplicating a pipeline **creates a copy** **within** the **same sub-account**.

  


  1. Go to **Opportunities** > **Pipelines**  
  

  2. Find the pipeline you want to duplicate and click the **three-dot menu** on the right and select **Duplicate**.  
  
![](https://jumpshare.com/share/mJPEKAEkS8TZRmLCvkzP+/Screen+Shot+2026-04-14+at+18.55.02.png)  
  

  3. A **Duplicate Pipeline** window will appear.  
  

  4. Update the **Pipeline Name:** Enter a new name for the duplicated pipeline. A default name will appear with “(copy)” added.  
  

  5. **Set pipeline display colors:** Choose how stage colors appear across your pipeline views.  
  

  6. **Modify Pipeline Stages:** Rearrange, rename, add/remove stages.   
  

  7. **Choose show in report options:** Select whether each stage should appear in your default dashboard. Disabling all stages hides the pipeline from the dashboard.  
  
![](https://jumpshare.com/share/UhgM6olratSUK2ZujeTr+/GIF+Recording+2026-04-14+at+19.42.09.gif)  
  

  8. The following **elements** **will** **be** **copied** **automatically** :  
  

     * Pipeline stages
     * Chart visibility settings
     * Smart tag colors
     * Pipeline permissions  
  

  9. The following **will NOT be copied** :  
  

     * Opportunity records
     * Automations
     * Reports
     * Custom fields  
  

  10. Click **Duplicate** to create the new pipeline.  
  
![](https://jumpshare.com/share/S6sxzNLbXJ4duiNxQZ4t+/Screen+Shot+2026-04-14+at+19.44.50.png)  
  

  11. The duplicated pipeline will appear in the pipeline list. You can now edit the pipeline or start using it immediately.  
  
![](https://jumpshare.com/share/VJgNxxngEUikwjjXjRyK+/Screen+Shot+2026-04-14+at+19.45.20.png)

## **How to Copy a Pipeline to Other Sub-Accounts**

  


This feature allows **Agency Admins** to copy a pipeline to multiple sub-accounts at once.

  


  1. Go to **Opportunities > Pipelines.**  
  

  2. Find the pipeline you want to copy and open the **three-dot menu** and select **Copy to sub-accounts**.  
  
![](https://jumpshare.com/share/RQaSWpniymmX66xsJ20A+/Screen+Shot+2026-04-14+at+19.49.53.png)  
  

  3. In the **Copy to Sub-Accounts** window, enter the following details.  
  

  4. **Pipeline Name:** You can keep the same name or change it.  
  

  5. **Select Sub-Account(s):** Choose one or more sub-accounts where the pipeline should be created.  
  

  6. Click **Copy** and the pipeline will be created in each selected sub-account.  
  
![](https://jumpshare.com/share/bEYV4PLvhhUQS8HCGPeT+/Screen+Shot+2026-04-14+at+19.52.39.png)

## **Best Practices**

  


A few simple habits can make pipeline duplication and distribution easier to manage at scale. These recommendations help reduce confusion, improve organization, and make future updates easier.

  


  * Rename duplicated pipelines immediately so they are easy to distinguish from the original.  
  


  * Use duplication to test pipeline variations before rolling them out more broadly.  
  


  * Standardize your preferred pipeline templates before copying them to other sub-accounts.  
  


  * Review which settings do and do not copy so you can complete any needed setup afterward.

## **Frequently Asked Questions**

  


**Q: Can I duplicate a pipeline inside the same sub-account?  
**

Yes. Use the **Duplicate** option from the three-dot menu next to the pipeline.  
  


**Q: Can I copy a pipeline to more than one sub-account at the same time?**  
Yes. You can select multiple destination sub-accounts in a single copy action.  
  


**Q: Who can copy pipelines to other sub-accounts?**  
Only **Agency Admins** can use the **Copy to sub-accounts** option.  
  


**Q: Who can duplicate a pipeline?**  
Any user with **Edit access** to that pipeline can duplicate it.  
  


**Q: Are opportunity records copied with the pipeline?**  
No. Records are not copied.  
  


**Q: Are automations, reports, or custom fields copied?**  
No. Automations, reports, and custom fields are not copied.  
  


**Q: Do smart tags copy with the pipeline?**  
Smart tag colors are copied when duplicating within the same sub-account, but they are not copied to other sub-accounts.  
  


**Q: Do pipeline permissions copy with the pipeline?**  
Pipeline permissions are copied when duplicating within the same sub-account, but they are not copied to other sub-accounts.  
  


**Q: Why do I not see the Copy to sub-accounts option?**  
That option is available to **Agency Admins only**.
