# **What is Labs?**

  


Labs is HighLevel’s early-access program where selected features can be turned on before they’re released to everyone. Features in Labs may change as development continues, and your feedback helps shape final behavior and timelines. Labs includes items that apply to Agency accounts and items that apply to sub-accounts.

  


Understanding Labs helps you evaluate upcoming changes in a controlled way. By enabling features in a test environment, validating impact, and sending feedback, you reduce risk during rollouts and influence product direction.

* * *

## **Key Benefits of Labs**

  


These benefits focus on how Labs improves your day-to-day operations, speeding up your ability to adopt improvements while safeguarding production sub-accounts.

  


  * **Early Access:** Try upcoming features before general release to plan processes and training.  
  

  * **Targeted Rollout:** Enable only for selected sub-accounts to run controlled pilots.  
  

  * **Faster Feedback Loops:** Send direct feedback (with screenshots/video) to inform product improvements.  
  

  * **Change Readiness:** Update SOPs and assets ahead of General Availability to minimize disruption.  
  

  * **Risk Reduction:** Validate behavior, performance, and compatibility before enabling for all clients.

# **Agency Labs vs Sub-Account Labs**

  


### ** _Agency Labs_**

  


  * **Affects:** Agency and sub-accounts (via assignment) or specific sub-accounts you choose.  
  


  * **Who can enable:** Agency Owners/Admins.  
  


  * **Where:** Agency view → **Settings → Labs** (Agency / Sub-account tabs).  
  


  * **How it rolls out:** Toggle **ON** at Agency level → **Manage** → assign one or more sub-accounts.  
  


  * **Best for:** Central governance, controlled pilots across multiple sub-accounts, global policies/tools.  


  


### **_Sub-Account Labs_**

  


  * **Affects:** Only the current sub-account; no impact on others.  
  


  * **Who can enable:** Sub-account Admins (and Agency roles when acting inside the sub-account).  
  


  * **Where:** Sub-account view → **Settings → Labs**.  
  


  * **How it rolls out:** Toggle **ON** directly in that sub-account (no assignment needed unless required by the feature).  
  


  * **Best for:** Quick, low-risk trials; team/client-specific validation.

## **How to Interpret Labs Features**

  


Every Labs feature appears as a “card.” Knowing each element helps you enable features confidently and communicate timelines to stakeholders.

  


Each card commonly includes:

  


  1. **Title:** Feature name.  
  

  2. **Description:** What changes or capabilities the feature provides.  
  

  3. **Live in X Days:** Estimated days until general availability (subject to change).  
  

  4. **Toggle ON/OFF:** Enable/disable the feature in the current context.  
  

  5. **Submit Feedback:** Send comments, issues, or requests directly to the product team.

## **Visibility vs Enablement Controls (Sub-Account Labs features)**

  


Labs controls use two different settings:

  


  1. Visibility: Whether a Labs feature appears in Sub-account view → Settings → Labs.  
  

  2. Enabled: Whether the feature is turned ON for that sub-account.  
  


    
    
    **Important:****Unlimited and Pro** **Plans** support visibility and enablement defaults and per-sub-account visibility and enablement management. 
    **Starter Plan** keeps Labs features visible by default and does not show visibility or enablement controls.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064178284/original/_lyXouZNCWkLXYkZks0wq-x1eYYU5ZnzTg.png?1770209216)

  


A feature can be visible but not enabled. This helps you introduce features for discovery without activating them.

  


### **Default visibility and overrides (Agency view)**

  


On eligible plans, agency admins can:  
  


  * Set the default visibility for new Labs features (Visible or Hidden).
  * Override visibility per sub-account when needed.

## **How To Use Labs**

  


Using Labs in the correct workspace is essential. Agency view exposes Agency-level features and tools for assigning sub-account features, while sub-account view shows features you can enable locally.

  


### _**1:** Enable an Agency‑only feature (Agency view)_

  


  1. Login to your**Agency** account**.**  
  

  2. Navigate to **Settings** → **Labs** → **Agency** **tab**.  
  

  3. Locate the feature card and review the **Description** and **Release** **Timeline**.  
  

  4. Toggle **ON**.  
  


![](https://jumpshare.com/share/txIBztOpI4GLQo1U1lUA+/GIF+Recording+2025-08-12+at+12.03.52+AM.gif)  


  


### _**2:** Enable a Sub‑account feature via Agency assignment _

  

    
    
    **Note:** Some sub-account features must be granted by the Agency before they can be used. The Manage control lets Agency Admins/Owners choose which sub-accounts get early access.

  


  1. Login to your **Agency** account.  
  

  2. Go to **Settings** → **Labs** → **Sub-account** tab.  
  

  3. Locate the feature card and review the **Description** and **Release** **Timeline**.  
  

  4. Click **Manage** and **select the sub-account(s)** to give access for Visibility and/or Enable.  
  

  5. Click **Save**.  
  

  6. Switch to the assigned sub-account(s) and refresh the relevant area to confirm the feature appears and/or is Enabled.


  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064179682/original/ypYnjb8wnXAoH9HBaTUlEcCaU15WlA0ACw.gif?1770209920)

  


  


### _**3:** Enable a Sub‑account feature directly_

  


Some sub-account features can be enabled directly within the sub-account without Agency assignment.  
  


  1. In the target sub-account, open **Settings** → **Labs**.  
  

  2. Find the feature card and toggle **ON**.  
  

  3. Refresh or re-open the relevant module (builder, CRM, etc.) to see changes.

## **Frequently Asked Questions**

  


**Q: Who can access Labs?  
** All Agency Admins and Sub-account Admins, plus any users who have the **View & Manage Settings** permission.

  


**Q: I don’t see Labs—what should I check?  
** Verify you’re in the correct context (Agency vs. sub-account) and that your role has permission to access Settings. If you still don’t see Labs, contact your account admin.

  


**Q: Can I disable a Labs feature later?****  
**Yes, you can just toggle Off the feature from where it was enabled.

  


**Q: What does “Live in X days” actually mean?  
** It’s an estimated timeline to general availability and can change based on feedback and development.

  


**Q: Where should I report bugs vs. feature requests?  
** Use Submit Feedback on the card for product input and small issues. For urgent support needs that block your team, follow your standard support/ticket workflow.

  


**Q: Why don’t I see a specific Labs feature in a sub-account?**

A: On eligible plans, your agency may have set the feature to **Hidden** (visibility). Ask an Agency Owner/Admin to change the feature’s visibility, then refresh the sub-account.

  


**Q: Do all plans have visibility controls?**

A: No. **Unlimited and Pro** support visibility defaults and per-sub-account visibility management. **Starter** keeps Labs features visible by default and does not show visibility controls.
