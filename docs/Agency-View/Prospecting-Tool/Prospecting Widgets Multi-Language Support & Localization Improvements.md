# Prospecting Widgets: Multi-Language Support & Localization Improvements

**Source URL:** [https://help.gohighlevel.com/support/solutions/articles/155000007361-prospecting-widgets-multi-language-support-localization-improvements](https://help.gohighlevel.com/support/solutions/articles/155000007361-prospecting-widgets-multi-language-support-localization-improvements)  
**Category:** Agency View  
**Folder:** Prospecting Tool

---

Prospecting Widgets now fully support multi-language functionality. All system-generated text — including field labels, validation messages, submit buttons, email communication, and PDF report links — automatically adapts to the user’s preferred platform language. This ensures a consistent localized experience across embedded widgets, emails, and exported reports.

## **Overview**

  


We’ve enhanced localization (i18n) support across the Prospecting Tool.

  


  


Previously, some backend-generated strings did not dynamically adapt to the user’s language preference. With this update:

  


  * Prospecting widgets display in the visitor’s preferred platform language  
  

  * Email communication respects user language settings  
  

  * PDF report links and exported content follow platform language preferences  
  

  * No additional configuration is required


  


This improvement ensures consistent multilingual behavior across customer-facing components.

## **Multi-Language Support**

  


Prospecting Widgets now fully support multiple languages.

  


  


All system-generated text automatically respects the user’s preferred platform language.

  


  


This includes:

  


  * Field labels  
  

  * Validation messages  
  

  * System prompts  
  

  * Submit button text  
  

  * Error messages  
  

  * Confirmation messaging


  


No manual translation setup is required.

## **Language Support Across Emails**

  


System-generated emails related to Prospecting Widgets now automatically adapt to the recipient’s platform language.

  


  


This includes:

  


  * Subject lines  
  

  * Call-to-action buttons  
  

  * System instructions


  


No additional configuration is required.

## **Frequently Asked Questions**

  


**Q: Do I need to manually enable multiple languages for widgets?**  
No. Widgets automatically adapt to the platform language.  
  
  
**Q: Does this change custom text I manually enter?**  
No. Only system-generated strings are localized.  
  
  
**Q: Are email notifications now multilingual?**  
Yes. System-generated emails follow the user’s preferred platform language.  
  
  
**Q: Do exported PDF reports change language automatically?**  
Yes. PDF report links and system text reflect platform language settings.  
  
  
**Q: Does this require upgrading the Prospecting Tool?**  
Yes. Widgets require the paid version.
