# **What is AI Studio**

  


AI Studio is a conversational, AI-powered builder that helps you turn ideas into working digital experiences faster. It can generate layouts, content, images, and multi-page experiences from a prompt, then lets you refine the result through follow-up instructions.

  


AI Studio can create one page or several connected sections in the same web project, helping users get to a more complete starting point faster instead of building each part manually. It also applies a consistent look across the page, generates content based on the prompt and page goal, and gives users a strong draft they can refine later. If a user does not provide images, AI Studio can generate visuals for hero sections and supporting areas. If the user uploads images and clearly asks AI Studio to use them, it can place those visuals into the page.

  


AI Studio is not limited to standard websites or landing pages. It can also be used to create interactive front-end experiences such as multi-step surveys, custom calendar booking flows, branded forms, and other client-facing experiences. This makes AI Studio a broader creative workspace for building customer experiences, not just static pages.

  


AI Studio can help you:  
  


  * Create a landing page using a live URL as inspiration for the layout and structure  
  

  * Build a storefront using your uploaded branded photos  
  

  * Generate a dashboard layout for a specific business use case  
  

  * Create an appointment booking page connected to your calendar  
  

  * Build a website with content, visuals, and calls to action already in place


  

    
    
    **Note:** AI Studio and Agent Studio are two separate AI tools within HighLevel, designed for different use cases.
    
    **AI Studio** is a conversational, AI-powered builder that helps you quickly create websites, pages, and other web assets using prompts. **[Agent Studio](<https://help.gohighlevel.com/en/support/solutions/articles/155000007393>)** is a visual builder used to create AI-powered agents that can respond to events, interact with users, and automate tasks.

## **Key Benefits of AI Studio**

  


AI Studio helps teams create faster, experiment more easily, and move from idea to launch with less manual work. Its biggest advantage is that it combines generation, iteration, previewing, and project management in one workspace.

  


  * **Faster Creation:** Start with a prompt instead of building every section manually  
  


  * **Natural Editing:** Request changes in plain language and let AI update the project  
  


  * **Visual Refinement:** Click elements in the live preview to update text, images, icons, spacing, colors, and layout details without editing code.  
  


  * **Flexible Inspiration:** Use templates, URLs, screenshots, and uploaded images to guide the output  
  


  * **Safe Iteration:** Use version history and restore options to test changes with more confidence  
  


  * **Built-In Organization:** Manage projects, folders, recent work, and starred items from one place  
  


  * **Production Path:** Preview, publish, and connect a custom domain when the project is ready

## **Creating a Project in AI Studio**

  


AI Studio gives you several ways to begin, which helps different users start from the type of input they already have. Whether you begin with an idea, a template, a website reference, or an image, the goal is the same: get to a strong first draft faster.

  


  


### **Start with a Prompt**

  


You can type a prompt that describes what you want to build. The more detail you provide, the better AI Studio can shape the first version. Along with your prompt, you can also use supporting inputs such as files, images or a live URL for structure, layout or style inspiration.

  


**Example prompts:**  
  


  * Create a website for a dental clinic with online booking  
  


  * Build a portfolio site for a freelance designer using the attached brand images  
  


  * Build a family campground homepage using this URL as inspiration for the navigation style and overall structure: https://www.campingvacation.com/  
  


  * Create a boutique skincare website using the attached homepage screenshot as visual direction for the hero section, spacing, and typography


  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068189083/original/HEJngiS-3EpUYojRGmY-HV92idJLzginGw.png?1774983033)

  


  


### **AI Studio Asks Follow-Up Questions When Needed**

  


Users do not need to write a perfect prompt. If the request is too broad, AI Studio can ask simple follow-up questions before building.

  


For example, if a user says _“build a website for my business”_ , AI Studio may ask what kind of business are we building this for? This makes the experience easier because the user can choose the goal instead of figuring out the setup alone.

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068189307/original/zgSJAj7ADqhj2ukVFbgBfO9zj8BqAlbqrw.png?1774983385)

  


  


### **Start with a Template**

  


Templates are useful when you want a faster starting point with a familiar structure. AI Studio offers templates for common project types like:

  


  * Landing page  
  


  * Dashboard  
  


  * Ecommerce Storefront  
  


  * Portfolio


  


To get started with a template, click the **Templates** tab then select the desired template.

## **How to Edit and Refine Projects**

  


AI Studio works best as a conversational builder. You are not locked into the first version it creates, which makes it easier to keep improving the result until it matches your goal. After generation, you can continue prompting AI Studio to make changes such as:  
  


  * Change colors or styling  
  


  * Rewrite headlines or body copy  
  


  * Add a contact form or booking section  
  


  * Use a reference URL or screenshot for a specific section


  


**Examples of follow-up prompts:**

  


  * Change the colors to use light green instead of orange  
  

  * Add a Contact page  
  

  * Make the top navigation similar to the reference URL  
  

  * Add a testimonial section similar to the attached screenshot  
  

  * Change the hero section visual to the attached image


  


AI Studio can also use uploaded images in the project. If you do not provide images, it may generate visuals for you.

## **Visual Edits in AI Studio**

  


Visual Edits gives users a direct way to refine AI Studio projects from the live preview. Instead of using prompts or opening the Code Editor for every small adjustment, users can select elements on the page and make visual updates to text, buttons, images, icons, colors, spacing, and layout details.

  


To enter Visual Edit mode, click **Visual Edits** in the prompt area at the bottom-left of the builder.

  


After Visual Edit mode is enabled, the preview becomes interactive. Users can double-click text to edit it inline, select elements to open contextual controls, or use a prompt on selected elements when they want AI help with a targeted change.

## **Advanced SEO Support in AI Studio**

  


AI Studio includes built-in **SEO tools** to help your projects get discovered by search engines, social platforms, and AI-powered search experiences.

  


Because AI Studio sites are built as fast, modern web apps, they can sometimes be harder for search engines and crawlers to read by default. Advanced SEO Support solves this by making your content more accessible and easier to index—without changing how your site looks or performs for visitors.

  


With SEO enabled, you can:  
  


  * Improve how your site is **crawled and indexed** by search engines  
  

  * Ensure shared links display the correct **title, description, and preview image**
  * Generate a **sitemap** to help search engines discover your pages faster  
  

  * Make your content more readable for **AI search tools** like ChatGPT and other assistants


  


These features work together to improve visibility across traditional search, social sharing, and emerging AI-driven discovery.

  


For step-by-step instructions and a full breakdown of SEO features, see: [Advanced SEO Support in AI Studio](<https://help.gohighlevel.com/en/support/solutions/articles/155000007691>)

## **Forms and Calendar Integration**

  


Forms and calendars help turn an AI-generated page into a working experience that can collect information or book time with your business. AI Studio can build the front-end experience for these actions then guide you through the connection steps needed to make them functional inside your HighLevel account. This makes the output more usable from the start while still letting you control when forms and calendars are connected.

####   

    
    
    **Important:** Forms and calendars are not connected automatically. AI Studio can create the layout for a form or booking section, but you need to ask it to connect that component before it starts sending data into your sub-account.

####   


### **How Forms Work in AI Studio**

  


AI Studio can create form-based sections and page experiences from a prompt. For example, you can ask it to add a contact form, registration form, feedback form, or lead capture section to the page. At this stage, the form starts as a front-end layout first. That means the form can appear on the page before it is connected to your CRM.

  


When you want the form to start collecting submissions, ask AI Studio to connect it or click **Connect** when prompted in the chat. This triggers the **Connect to CRM** flow, which is used to:  
  


  * Create contacts in the sub-account  
  


  * Capture form submissions  
  


  * Support workflows through tracking triggers


  


Once that connection step is completed, AI Studio can wire the form into the connected setup so submissions flow into your HighLevel account.

  


For a full walkthrough, see the dedicated article on [connecting forms and calendars in AI Studio.](<https://help.gohighlevel.com/en/support/solutions/articles/155000007599>)

  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068191404/original/FyiF8GgFRB8sH5aMjswil-6JEZRrZvySSw.png?1774986115)

  


  


### **How Calendars Work in AI Studio**

  


AI Studio can also create booking-focused pages. If you ask for an appointment page, consultation booking page, or discovery call page, it can build the page around scheduling and add a booking section.

  


To make that booking experience functional, AI Studio can detect calendars that already exist in the sub-account and prompt you to choose which one to connect. At a high level, the calendar flow works like this:

  


  1. AI Studio creates the booking area on the page.  
  


  2. You select a calendar from the sub-account.  
  


  3. That calendar is connected to the page.


  


This makes it easier to turn a booking page into a working scheduling experience without rebuilding it manually.

  


For a full walkthrough, see the dedicated article on [connecting forms and calendars in AI Studio.](<https://help.gohighlevel.com/en/support/solutions/articles/155000007599>)

## **Publishing Your Projects**

  


Publishing your AI Studio project makes it available through a live URL so you can review it, share it, and launch it on your own domain. HighLevel supports a simple publishing flow that starts with a preview URL, then lets you connect a custom domain and choose the main published URL visitors should use.

  


After a site is published, users can continue making edits. Those changes remain in draft until the user publishes again. This makes it safer to improve the page without changing the live version right away.

  


When an AI Studio project is connected to a domain:  
  


  * The project lives in **AI Studio** under Sites  
  


  * The domain connection is visible in **Domains**  
  


  * The published page appears on that connected domain


  


  


### **Publish to a Preview Domain**

  


Publishing to a preview domain gives you a live version of your AI Studio project before you connect a branded domain. This helps you test the experience, review the layout, and confirm everything works as expected before going live on your custom domain.

  


To publish your project to a preview domain:

  


  1. Click **Publish** in the top right corner. Update the URL if needed, then click **Publish** again.

  2. Update the icon, name, description, and social image. Then click **Apply Changes.**  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068821959/original/kgClo-r43ZlQ-Woe9YuIk0JhLhKwXNqHsA.jpeg?1775752289)  
  

  3. Click **Publish**.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068822182/original/sf-3VfJsNIZgyrUA86lAM0EFrAUeO_FA-A.png?1775752490)


  


  


### **Connect a Custom Domain**

  


Connecting a custom domain lets you publish your AI Studio project on a branded URL. You cannot connect a custom domain until the project has first been published to a preview domain.

  


To connect a custom domain:

  


  1. Click **Publish** then select **Add Custom Domain**.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068835459/original/H2V8S-e_-JaTDCcvsUXUxNUSVpJSrQarMg.jpeg?1775767251)  
  


  2. Enter the domain you want to connect.  
  
If you enter an apex domain (example.com) or the www version (www.example.com), an option appears to also add the other domain and set up a redirect. It is recommended to connect both your root and www domains.  
  
Leave it selected to connect both domains in the same setup. Clear it if you only want to connect the domain you typed. This option only appears for apex and www domains (not for other subdomains like blog.example.com).  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155070409908/original/rTCG2ESLDAgHhLCCjULYhYbzTQOo6dKv3w.png?1777669854)  
  


  3. If the domain is already managed in HighLevel, the setup continues within the flow. If conflicting records are found, review the warning and affected records before clicking **Add Records**.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068835982/original/OEJK6ZRmM0mhKZX8r2aYcdrLXFLYxJNvaQ.png?1775768493)  
  


  4. If the domain is not managed in HighLevel, add the DNS record(s) shown, then click **Verify DNS**. If you chose to connect both apex and www, you’ll be prompted to add/verify two records in the same flow.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155070410190/original/MtxSW9cie-fOn-UiXjlJtvnrMt5DU37Hfg.png?1777670254)  
  


  5. After setup is complete, the AI chat shows that the custom domain is connected, displays the connected domain, and includes a **View site** option.

  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155070410386/original/DGxwASNmGBE1PcoxTEE2K0pNlJqmXCbvnQ.png?1777670679)


  


  


### **Set a Primary Published URL**

  


Setting a primary published URL lets you choose the main live domain for your AI Studio project. Other connected URLs will automatically do a 301 redirect to the primary domain. 

  

    
    
    **Note:** If you connected both the apex and www domains during setup, the domain you typed is treated as the primary (canonical) URL and the other domain redirects to it. If your project already has a primary URL set, it remains unchanged.

  


To set a primary published URL:  
  


  1. Click **Publish** , then click the three-dot menu next to the URL you want to make primary.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068836276/original/1siC6pzPEv1WqOaV6Ot99pSnCQInzlgfPQ.png?1775769184)  
  

  2. Click **Set as primary**.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068836300/original/s-4UgjNUBa-SmZn1Pl6xufKCsVisIZjUVA.png?1775769254)

## **Cloning Projects in AI Studio**

  


  


  


AI Studio lets you clone a project within the same sub-account or across sub-accounts in the same agency. Project cloning makes it easier to reuse successful work without rebuilding the same experience from scratch.

  


When cloning a project, you can:  
  


  * Duplicate it inside the same sub-account  
  

  * Clone it to another sub-account in your agency  
  

  * Place it at the root level or inside an existing folder  
  

  * Choose whether to include chat activity and version history from the original project


  


  


Follow the steps below to clone a project:

  


  1. Click the **three-dot menu** on the project you would like to clone and select **Clone**.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068924332/original/PZ-FckJc7-LVa8fix18LhRbC3qTqxRKOqw.png?1775854064)  
  

  2. In the **Clone project** window, enter the new project name. Choose the target sub-account and target folder(or leave it at the root level).  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068924267/original/D6XOyKOQpNCpwurAghxi6GbyfYEyvI2Bxg.png?1775853863)  
  

  3. Turn **Include project history** on or off. **With History** includes chat activity and version history from the original project. **Without History** creates a clean copy with no chats or version history.  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068924313/original/IcecIuNzkFD6kwrUYpNIpnU3YSgF1BQZow.png?1775854009)  
  

  4. Click **Clone**.

## **Submission Data of your Project**

  


Once connected, form submissions go into the connected sub-account CRM setup, not into the AI Studio project itself. This means that the page lives in AI Studio but the responses, contacts and submission activity go into the connected HighLevel account. After connection, form submissions can appear under form submissions as third-party submissions, and they can also be tied into workflow activity through external tracking behavior.

  


Users can view submissions in:  
  


  1. Contacts  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068402610/original/WG8p0Qv6vvip67qp9vWFhANsCaBM0ko5BA.jpeg?1775194728)  
  

  2. Sites > Forms > Submissions > External Forms  
  
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068402612/original/k78YNSw2jXZCf6XxEWqkH3TA0MDN93SCaw.jpeg?1775194728)


  


  


### **How Workflows Connect to Form Submissions**

  


Once the form is connected to CRM tracking, submissions can be used in workflows through tracking triggers. The practical flow is:

  


  1. Create the form layout in AI Studio  
  

  2. Connect the form to CRM tracking  
  

  3. In a workflow, use the trigger "External Tracking Event"  
  

  4. Use the "Form submission" event to continue automations  
  

  5. Use the "Domain" filter and/or "External Form" filter then continue your automation actions.


  


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068402611/original/8MOyTIdDTYJpOaGpgIYVxkMME7NWSw_zIg.jpeg?1775194728)

## **Current Limitations and Important Notes**

  


Knowing the current boundaries of AI Studio helps set expectations and prevents confusion during setup. Since AI Studio is available in Labs, some behavior may continue to evolve over time.

  


**Important notes:**  
  


  * AI Studio projects are managed inside AI Studio and cannot be moved into the standard Sites or Funnels builders.  
  


  * URL and image-based recreation should be treated as inspiration for structure, layout, and style direction rather than an exact one-to-one copy.

## **Frequently Asked Questions**

  


**Q: Is AI Studio only for websites and landing pages?**  
No. AI Studio can also create other front-end experiences such as multi-step surveys, branded forms, booking flows, and other interactive customer-facing experiences.

  


**Q: How much does AI Studio cost?**  
AI Studio is currently free and pricing is expected to evolve over time.

  


**Q: Do I need to move an AI Studio project into Funnels or Websites before publishing it?**  
No. AI Studio projects can be published directly from AI Studio. You can build multiple pages, connect them within the same project, and then publish and connect a domain from there

  


**Q: Can I move or copy an AI Studio project into Funnels or Websites under Sites?**  
No. AI Studio projects stay in AI Studio and cannot be not copied into the standard Funnels or Websites builders under Sites. AI Studio is designed to publish directly from its own workspace, you do not need to move the output into the legacy Funnels or Websites builders.

  


**Q: Can I preview my project on mobile before publishing?**  
Yes. AI Studio includes preview options for desktop, tablet, and mobile views so you can review the layout before publishing. Just click on the Computer icon in the top of the preview to switch between view modes
